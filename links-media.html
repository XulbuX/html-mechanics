Link Functions

  • Linking to another webpage: Google (this tab) 
    <a href="https://www.google.com" target="_self">Google (this tab)</a>
  • Linking to a text passage in the same document. This text passage must also be defined with an anchor. The anchor has an id attribute with a unique name:
    (Example: to the Media section on this page)
    <a href="#internal-link-target">Jump to target anchor</a>

<!-- ... later in the document ... -->
<h2 id="internal-link-target">Target anchor for internal link</h2>
  • Linking an email address, so the user can directly write a new email: example@domain.com 
    <a href="mailto:example@domain.com">example@domain.com</a>
  • Linking a phone number, so this number can be called directly on mobile devices: Call us: +01 234 567 89 
    <a href="tel:+0123456789">Call us: +01 234 567 89</a>
  • Additionally, links can be programmed to open in a new tab: Google (new tab) 
    <a href="https://www.google.com" target="_blank">Google (new tab)</a>

Image Formats for the Web

  • GIF: lossily removes a large part of the colors from an image, leaving 255 different colors. Transparency is either fully transparent or completely opaque.
    Typical usage: short loops, reaction GIFs, simple logos or icons where maximum compatibility is important.
  • JP(E)G: the lossy compression can be specified with a value between 0 and 100.
    100 = low compression > large file
    0 = high compression > small file
    Typical usage: photos, photorealistic backgrounds, social media images, product photos in shops.
  • PNG: lossless image format, therefore rather large file, but very detailed. Supports transparency with an alpha channel.
    Typical usage: UI elements, screenshots, logos & icons with transparency, diagrams, pixel art.
  • AVIF: newer image format, offers lossless and lossy compression.
    Is ~50% smaller than a JPEG of similar quality, supports transparency with an alpha channel.
    Typical usage: modern websites with many photos / HDR images, e-commerce, animated stickers with transparency.
    caniuse.com/?search=avif
  • WEBP: newer image format, offers lossless and lossy compression, supports transparency with an alpha channel.
    Typical usage: general web graphics, photos, thumbnails, animated stickers; good compromise between file size & support.
    caniuse.com/?search=webp
  • SVG: Scalable Vector Graphic - Vector-based image format, information is described with coordinates, can be scaled losslessly. Very small file size. Not suitable for photos.
    Typical usage: logos, icons, diagrams, infographics, UI illustrations, responsive graphics.

Embedding Images on a Webpage:

Simple embedding:

<img src="assets/img/notexistent.jpg" alt="Not existing image">

Example:

The <picture> tag is used to provide different formats or sizes depending on the browser or screen size - ideal e.g. for modern formats like WebP or AVIF with fallbacks:

<!-- Embedding images on a webpage with <picture> -->
<picture>
  <!-- Modern formats first -->
  <source srcset="assets/img/image.avif" type="image/avif">
  <source srcset="assets/img/image.webp" type="image/webp">
  <!-- Fallback for browsers that do not support AVIF/WebP -->
  <img src="assets/img/image.jpg" alt="Image description">
</picture>

Example (the browser chooses the best supported format):

  • The <source> elements specify alternative formats. The browser selects the first supported format.
  • The <img> element serves as a fallback.

Further Information:

  • MDN Web Docs: <img>: The Image Embed element
  • MDN Web Docs: <picture>: The Picture element

Video on the Web

Embedding a video is done using the <video> tag. It's recommended to provide multiple source formats for broader browser compatibility.

<video controls>
  <source src="assets/video/example.mp4" type="video/mp4">
  <source src="assets/video/example.webm" type="video/webm">
  Your browser does not support the video tag.
</video>

Example (the browser chooses the best supported format):

  • The controls attribute adds video controls, like play, pause, and volume.
  • The width attribute sets the width of the video player.
  • The poster attribute specifies an image to be shown while the video is downloading, or until the user hits the play button.
  • <source> elements specify alternative video files. The browser will choose the first source it supports.
  • Other useful attributes include autoplay, loop, and muted.

Further Information:

  • MDN Web Docs: <video>: The Video Embed element
  • MDN Web Docs: Common video codecs and containers

Audio on the Web

Embedding audio is done using the <audio> tag. Similar to video, providing multiple source formats is a good practice.

<audio controls>
  <source src="assets/audio/example.mp3" type="audio/mpeg">
  <source src="assets/audio/example.ogg" type="audio/ogg">
  Your browser does not support the audio element.
</audio>

Example (the browser chooses the best supported format):

  • The controls attribute adds audio controls, like play, pause, and volume.
  • <source> elements specify alternative audio files. The browser will choose the first source it supports.
  • Other useful attributes include autoplay, loop, and muted.
